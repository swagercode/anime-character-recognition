# Learning Log

- A Git repository is a project folder tracked by Git.
- A Git remote is a named URL, commonly `origin`, where commits can be pushed or pulled.
- A Git commit is a saved snapshot of tracked file changes.
- A clean working tree means there are no uncommitted tracked changes.
- `python -m venv .venv` creates a local virtual environment.
- A virtual environment keeps this project's installed Python packages separate from other projects.
- `.venv` should not be committed because it is generated environment state.
- `pyproject.toml` stores Python project metadata and tool configuration.
- An editable install lets Python import the package directly from the source tree while developing.
- A Python module is usually a single `.py` file.
- A Python package is a directory of importable Python modules, usually with `__init__.py`.
- The project uses a `src` layout so package code stays separate from tests and project config.
- `pytest` only proves the code paths that the tests actually execute.
- A project can pass tests while still having a runtime bug if the broken path is not tested.
- A CLI is a command-line interface.
- Python's standard-library `argparse` module parses command-line arguments.
- `argparse.ArgumentParser.parse_args()` returns an `argparse.Namespace`.
- In `argparse`, `dest` controls the attribute name used on the returned `Namespace`.
- A command-line option like `--image` becomes an attribute like `args.image`.
- A subcommand is a command inside a CLI, such as `classify`.
- A `NameError` happens when Python cannot find a requested name.
- A scope is a region where Python can look up names.
